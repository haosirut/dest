# Биллинговая система VaultKeeper P2P

## Структура тарифов

### Базовая ставка

Базовая стоимость хранения составляет **0,30 руб. за 1 ТиБ в час**. Тариф применяется к фактическому объёму занятых данных с учётом репликации.

```
Базовая ставка = 0,30 RUB / TiB / час
```

> 1 ТиБ = 1 099 511 627 776 байт (теибайт, двоичный префикс).

### Множители типа диска

Стоимость хранения зависит от типа диска, используемого хост-провайдером:

| Тип диска | Множитель | Обоснование |
|-----------|-----------|-------------|
| HDD | 1,0× | Базовая стоимость. Низкая скорость, но большое объём при низкой цене |
| SSD | 1,5× | Ускорённый доступ. Оптимально для часто используемых данных |
| NVMe | 2,0× | Максимальная скорость. Для критичных рабочих нагрузок |

Множитель применяется к базовой ставке:

```
Стоимость за TiB/час = Базовая ставка × Множитель диска × Множитель репликации × (1 + Подушка)
```

### Множители репликации

Пользователь выбирает уровень репликации, определяющий количество копий каждого шарда в сети:

| Репликация | Множитель | Мин. шардов для восстановления | Макс. потерянных шардов |
|-----------|-----------|-------------------------------|------------------------|
| 1x | 1,0× | 3 из 5 | 2 из 5 (40%) |
| 2x | 2,0× | 3 из 10 | 7 из 10 (70%) |
| 3x | 3,0× | 3 из 15 | 12 из 15 (80%) |
| 4x | 4,0× | 3 из 20 | 17 из 20 (85%) |

### Подушка (Cushion)

К итоговому тарифу добавляется **подушка 25%**. Подушка покрывает:

- Издержки на gossip-трафик и heartbeat-запросы
- Proof-of-Storage вычисления
- Резерв на форс-мажорные ситуации (массовый выход узлов)
- Маржу платформы (части от 10%)

```
Итоговый тариф = Базовая ставка × Множитель_диска × Множитель_репликации × 1,25
```

## Примеры расчётов

### Пример 1: Базовый сценарий

Пользователь хранит **100 ГиБ** данных на HDD с репликацией 1x.

```
Объём в ТиБ = 100 × 1024³ / 1024⁴ = 100 / 1024 = 0,09765625 TiB
Стоимость/час = 0,30 × 1,0 (HDD) × 1,0 (1x) × 1,25 (подушка) × 0,09765625
             = 0,30 × 1,25 × 0,09765625
             = 0,375 × 0,09765625
             = 0,03662 RUB/час

Стоимость/сутки = 0,03662 × 24 = 0,87891 RUB/сутки
Стоимость/месяц  = 0,87891 × 30 = 26,37 RUB/месяц (30 дней)
```

### Пример 2: SSD с тройной репликацией

Пользователь хранит **500 ГиБ** данных на SSD с репликацией 3x.

```
Объём в ТиБ = 500 / 1024 = 0,48828125 TiB
Стоимость/час = 0,30 × 1,5 (SSD) × 3,0 (3x) × 1,25 × 0,48828125
             = 0,30 × 1,5 × 3,0 × 1,25 × 0,48828125
             = 1,6875 × 0,48828125
             = 0,82397 RUB/час

Стоимость/сутки = 0,82397 × 24 = 19,77539 RUB/сутки
Стоимость/месяц  = 19,77539 × 30 = 593,26 RUB/месяц (30 дней)
```

### Пример 3: NVMe с максимальной репликацией

Пользователь хранит **2 ТиБ** данных на NVMe с репликацией 4x.

```
Стоимость/час = 0,30 × 2,0 (NVMe) × 4,0 (4x) × 1,25 × 2,0
             = 0,30 × 2,0 × 4,0 × 1,25 × 2,0
             = 3,0 × 2,0
             = 6,00 RUB/час

Стоимость/сутки = 6,00 × 24 = 144,00 RUB/сутки
Стоимость/месяц  = 144,00 × 30 = 4 320,00 RUB/месяц (30 дней)
```

## Распределение доходов

Доход от хранения распределяется между платформой и хост-провайдерами:

| Получатель | Доля | Назначение |
|-----------|------|-----------|
| Платформа | 10% | Развитие, инфраструктура, поддержка |
| Хост-провайдеры | 90% | Вознаграждение за хранение данных |

Распределение осуществляется автоматически при каждом heartbeat. Средства начисляются на счёт каждого хоста пропорционально объёму хранимых данных.

### Пример распределения

При стоимости хранения 100 RUB/месяц:

```
Платформа:       100 × 0,10 = 10,00 RUB
Хост-провайдеры: 100 × 0,90 = 90,00 RUB
```

Если данные распределены по 3 хостам равномерно:

```
Хост 1: 90,00 / 3 = 30,00 RUB
Хост 2: 90,00 / 3 = 30,00 RUB
Хост 3: 90,00 / 3 = 30,00 RUB
```

## Подписочные уровни

### Таблица уровней

| Параметр | Archive | Standard | Premium |
|----------|---------|----------|---------|
| Стоимость (RUB/мес) | 0 (бесплатно) | 199 | 499 |
| Включённый объём | 5 ГиБ | 500 ГиБ | 5 ТиБ |
| Репликация | 1x | 2x | 4x |
| Приоритет восстановления | Низкий | Средний | Высокий |
| Proof-of-Storage частота | Каждые 4 часа | Каждые 15 минут | Каждые 5 минут |
| Техническая поддержка | Сообщество | Email | Приоритетный Email + Telegram |

### Перерасчёт при смене уровня (Pro-rata)

При смене подписки в середине расчётного периода выполняется пропорциональный перерасчёт:

```
Использовано_дней = Текущая_дата - Дата_начала_периода
Остаток_дней = 30 - Использовано_дней
Стоимость_старого_за_день = Цена_старого / 30
Стоимость_нового_за_день = Цена_нового / 30

Кредит = Остаток_дней × Стоимость_старого_за_день
Дебет  = Остаток_дней × Стоимость_нового_за_день

Разница = Дебет - Кредит
```

Если Разница > 0 — списывается с баланса. Если Разница < 0 — зачисляется на баланс.

**Пример:** Переход со Standard (199 RUB) на Premium (499 RUB) на 10-й день месяца.

```
Использовано дней = 10
Остаток дней = 20
Кредит = 20 × (199 / 30) = 20 × 6,6333... = 132,67 RUB
Дебет  = 20 × (499 / 30) = 20 × 16,6333... = 332,67 RUB
Разница = 332,67 - 132,67 = 200,00 RUB (списывается)
```

Все расчёты выполняются с типом `rust_decimal::Decimal` (28 знаков точности) для исключения ошибок округления.

## Механизм заморозки

### Статусы аккаунта

| Статус | Условие | Действие |
|--------|---------|----------|
| **Активный** | Баланс > 0 | Полный доступ ко всем функциям |
| **Заморожен** | Баланс ≤ 0 | Чтение и экспорт данных в течение 48 часов |
| **Удалён** | Прошло 48 часов после заморозки | Данные удалены безвозвратно |

### Процесс заморозки

1. При heartbeat (каждые 15 минут) проверяется баланс аккаунта
2. Если баланс ≤ 0 и статус «Активный» → статус меняется на «Заморожен»
3. Уведомление отправляется пользователю (email/push)
4. Хост-узлы получают gossip-уведомление о заморозке
5. В течение 48 часов пользователь может:
   - Пополнить баланс → аккаунт размораживается автоматически
   - Экспортировать все свои данные (скачать)
6. Через 48 часов:
   - Если баланс > 0 → автоматическая разморозка
   - Если баланс ≤ 0 → данные помечаются к удалению
   - Хосты удаляют шард(ы), ledger удаляет записи

## Минимальный вывод средств

Минимальная сумма вывода для хост-провайдеров составляет **100 RUB**. Вывод выполняется на банковскую карту через платёжную систему.

| Параметр | Значение |
|----------|----------|
| Минимальный вывод | 100,00 RUB |
| Комиссия платформы | 0 RUB |
| Срок обработки | 1–3 рабочих дня |
| Способ вывода | YooKassa → банковская карта |

## Точность расчётов

Все финансовые расчёты в VaultKeeper P2P выполняются с использованием типа `rust_decimal::Decimal`:

- Точность: 28 значащих десятичных цифр
- Нет потери точности при умножении/делении (в пределах 28 цифр)
- Округление до копеек (2 знака) только при финальном списании/начислении
- Стратегия округления: MidpointAwayFromZero (0,5 округляется вверх)

Это полностью исключает классические проблемы с плавающей точкой, такие как `0,1 + 0,2 ≠ 0,3`.

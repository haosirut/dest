# Правовые аспекты VaultKeeper P2P

## Правовая структура сервиса

### Определение деятельности

**VaultKeeper P2P** — это сервис **координации одноранговой (P2P) сети хранения данных**. С юридической точки зрения платформа НЕ является хостинг-провайдером, облачным хранилищем или оператором персональных данных.

Ключевые правовые характеристики:

1. **Платформа** предоставляет программное обеспечение и инфраструктуру для координации взаимодействия между пользователями.
2. **Пользователи** (клиенты) шифруют данные на своих устройствах и самостоятельно управляют ключами.
3. **Хост-провайдеры** предоставляют дисковое пространство и вычислительные ресурсы для хранения зашифрованных шардов.
4. **Платформа** не имеет доступа к содержимому данных, не хранит ключей пользователей и не может расшифровать передаваемую информацию.

### Правовая модель

```
┌──────────────────┐         ┌──────────────────────┐
│   Платформа       │         │   Хост-провайдеры    │
│   (ООО "VK")      │◄───────►│   (НПД / ИП)         │
│                   │  Договор │   Агентский договор  │
│   Координация     │  агентск.│   или договор ГПХ   │
│   сети, биллинг,  │         │                      │
│   маркетплейс     │         └──────────────────────┘
└───────┬───────────┘
        │
        │  Лицензионное соглашение / OFERTA
        │
┌───────┴───────────┐
│   Пользователи    │
│   (физ. лица)     │
│   Шифрование,     │
│   хранение,       │
│   управление      │
└───────────────────┘
```

## Соответствие законодательству Российской Федерации

### Ключевые нормативные акты

| Закон | Сфера регулирования | Статус VaultKeeper P2P |
|-------|--------------------|-----------------------|
| 152-ФЗ «О персональных данных» | Защита персональных данных | ✅ Платформа не обрабатывает персональные данные пользователей (данные зашифрованы на клиенте) |
| 54-ФЗ «О применении контрольно-кассовой техники» | Кассовые чеки | ✅ Эквайринг через YooKassa (контактный платёжный агент), чеки формирует платёжная система |
| 149-ФЗ «Об информации, информационных технологиях...» | Информационные технологии | ✅ Регистрация в Роскомнадзоре как организатор распространения информации |
| 126-ФЗ «О конфиденциальности» | Тайна связи | ⚠️ Не применим (трафик P2P, но зашифрован) |
| 242-ФЗ «О внесении изменений...» | Локализация данных | ✅ Все данные зашифрованы и хранятся на территории РФ (у российских хостов) |

## Соответствие 54-ФЗ (ККТ)

### Статус

Платформа **не является расчётно-кассовым центром** и **не применяет контрольно-кассовую технику (ККТ)** напрямую.

### Модель через платёжного агента

Все платёжные операции проходят через **ООО «ЮKassa»** (партнёр платформы), который является банковским платёжным агентом и самостоятельно применяет ККТ. Платформа выступает как комиссионер/агент.

**Схема:**

1. Пользователь инициирует платёж через интерфейс VaultKeeper
2. Платёж перенаправляется на страницу YooKassa
3. YooKassa обрабатывает платёж, формирует кассовый чек по 54-ФЗ
4. YooKassa уведомляет платформу об успешном платеже
5. Платформа зачисляет средства на баланс пользователя

### Фискализация выплат хост-провайдерам

Выплаты хост-провайдерам также осуществляются через YooKassa. YooKassa формирует кассовые чеки при каждой выплате. Платформа не взаимодействует с ККТ напрямую.

## Соответствие 152-ФЗ (Персональные данные)

### Почему 152-ФЗ не применим к данным пользователей

1. **Клиентское шифрование**: Данные пользователей шифруются на устройстве с помощью XChaCha20-Poly1305. Ключ шифрования известен только пользователю и никогда не покидает его устройство.

2. **Отсутствие доступа**: Платформа не имеет технической возможности расшифровать данные. Даже при получении решения суда платформа не может предоставить содержимое файлов — ключей у неё нет.

3. **Зашифрованные шард(ы)**: На хост-провайдерах хранятся зашифрованные фрагменты (шард(ы)) данных. Без ключа шифрования и информации о распределении шардов невозможно восстановить файл.

4. **Метаданные**: Метаданные (имена файлов, размеры, даты) шифруются вместе с содержимым. Платформа хранит только идентификаторы файлов и объёмы.

### Данные, которые обрабатывает платформа

Платформа обрабатывает **минимальный набор технических данных**, необходимых для функционирования сервиса:

| Данные | Основание | Срок хранения |
|--------|-----------|---------------|
| Email пользователя | Договор оферты, уведомления | До удаления аккаунта |
| Баланс, история платежей | Договор оферты, налоговый учёт | 5 лет (налоговые) |
| IP-адрес (временный) | Безопасность, предотвращение злоупотреблений | 30 дней |
| Peer ID (ed25519) | Функционирование P2P-сети | Бессрочно |

Эти данные не являются персональными данными в контексте 152-ФЗ, так как они используются исключительно для технических целей и не позволяют идентифицировать конкретное лицо в сочетании с зашифрованными данными.

## Статус хост-провайдера

### Рекомендуемые формы

Хост-провайдер — физическое лицо, предоставляющее дисковое пространство в сети VaultKeeper. Рекомендуемые формы деятельности:

#### 1. Налог на профессиональный доход (НПД / Самозанятый)

Наиболее простая форма для хост-провайдеров:

- Ставка налога: 4% при работе с физлицами, 6% при работе с юрлицами
- Регистрация через приложение «Мой налог»
- Автоматический расчёт налога
- Налоговая декларация не требуется
- Лимит дохода: 2 400 000 RUB/год
- Нет обязательных страховых взносов

#### 2. Индивидуальный предприниматель (ИП)

Для хостов с большим объёмом:

- УСН 6% (доходы) или 15% (доходы минус расходы)
- Обязательные страховые взносы (минимум)
- Необходима отчётность
- Нет лимита на доход

### Договор между платформой и хостом

Платформа предоставляет типовой **агентский договор** или **договор ГПХ** для хост-провайдеров:

- Предмет: оказание услуг по хранению зашифрованных данных
- Вознаграждение: 90% от оплаты пользователей за хранение
- Выплата: через YooKassa на банковскую карту хоста
- Минимальный вывод: 100 RUB
- Срок выплаты: 1–3 рабочих дня

## Платёжная обработка

### Провайдер: YooKassa + СБП

Все платёжные операции обрабатываются через **ООО «ЮKassa»** (лицензия ЦБ РФ № 2784 от 11.09.2018).

#### Приём платежей от пользователей

| Способ | Комиссия | Время зачисления |
|--------|----------|-----------------|
| Банковская карта (Visa/MC/Мир) | 2,8% + 30 RUB | Мгновенно |
| СБП (Система быстрых платежей) | 0,4% | Мгновенно |
| ЮMoney | 0,5% | Мгновенно |

#### Выплаты хост-провайдерам

| Способ | Комиссия | Срок |
|--------|----------|------|
| На банковскую карту | 0 RUB | 1–3 рабочих дня |
| СБП | 0 RUB | Мгновенно |

### Схема работы

```
Пользователь           Платформа              YooKassa           Хост-провайдер
     │                     │                      │                    │
     │── Платёж 199 RUB ──►│                      │                    │
     │                     │── Запрос на платёж ─►│                    │
     │                     │                      │── Кассовый чек ──►  │
     │                     │◄── Подтверждение ────│                    │
     │◄── Активация ───────│                      │                    │
     │                     │                      │                    │
     │    (использование хранилища, 90% → хосту)  │                    │
     │                     │                      │                    │
     │                     │── Запрос на выплату ►│                    │
     │                     │                      │── 90 RUB ────────►│
     │                     │                      │── Кассовый чек ──►  │
```

## Автоматическая генерация документов

### Договоры с хост-провайдерами

Платформа автоматически генерирует и подписывает (ЭДО) следующие документы:

1. **Агентский договор / Договор ГПХ** — при регистрации хоста
2. **Акт оказанных услуг** — ежемесячно, на основании фактически хранимых данных
3. **Счёт на оплату** — при каждой выплате
4. **Отчёт агента** — ежеквартально (для налоговых целей)

### Процесс генерации

1. Система автоматически формирует документы на основании данных из billing-модуля
2. Документы подписываются ЭЦП платформы
3. Хост-провайдер получает документы в личном кабинете (Tauri GUI) или по email
4. При необходимости — подписание через Контур.Диадок или СБИС (ЭДО)

### Лицензионное соглашение с пользователями

- **Оферта** (публичная оферта) — доступна на сайте и в приложении
- Принятие оферты — при первой оплате подписки
- Политика конфиденциальности — отдельный документ
- Согласие на обработку данных — интегрировано в процесс регистрации

## Рекомендации по юридическому обеспечению

1. **Регистрация ИП/ООО** для оператора платформы — обязательна для заключения договоров с YooKassa и хост-провайдерами.
2. **Регистрация в Роскомнадзоре** как организатора распространения информации (если осуществляется маршрутизация информации пользователей).
3. **Юридическая консультация** — перед запуском рекомендуется получить заключение юриста по ИТ-праву относительно конкретной схемы работы.
4. **Политика AML/KYC** — базовая идентификация пользователей через YooKassa; усиленная — при выводе средств от 15 000 RUB/месяц (115-ФЗ).
5. **Соглашение SLA** — для Premium-подписчиков рекомендуется заключать индивидуальное SLA.

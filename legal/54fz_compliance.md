# СООТВЕТСТВИЕ ТРЕБОВАНИЯМ 54-ФЗ — VAULTKEEPER P2P

**Версия документа:** 1.0
**Дата вступления в силу:** 01 июля 2025 г.

---

## 1. ОБЗОР ТРЕБОВАНИЙ 54-ФЗ

1.1. Федеральный закон от 22.05.2003 № 54-ФЗ «О применении контрольно-кассовой техники при осуществлении наличных денежных расчётов и (или) расчётов с использованием платёжных карт» (далее — «54-ФЗ») устанавливает обязательные требования к использованию контрольно-кассовой техники (ККТ) и фискализации расчётов.

1.2. Основные требования 54-ФЗ, применимые к платформе VaultKeeper:

- Применение зарегистрированной ККТ при каждом расчёте с физическими лицами.
- Формирование кассового чека при приёме оплаты (как в электронной, так и в бумажной форме).
- Передача фискальных документов в налоговые органы через оператора фискальных данных (ОФД).
- Отправка электронного чека покупателю (потребителю) по SMS, электронной почте или через QR-код.
- Хранение фискальных данных в течение не менее 5 лет.

1.3. С 2024 года требования 54-ФЗ распространяются также на расчеты в электронных платежных системах, включая приём платежей через интернет-сервисы.

## 2. ПРИМЕНЕНИЕ 54-ФЗ К VAULTKEEPER

2.1. В контексте платформы VaultKeeper 54-ФЗ применяется в следующих сценариях:

- **Приём оплаты от Пользователей** за услуги хранения данных (оплата тарифов, покупка дополнительного пространства).
- **Выплата вознаграждений Хостам** — денежные средства, перечисляемые Хостам, также подлежат фискализации, поскольку Хосты являются потребителями платёжных услуг Платформы.

2.2. Платформа VaultKeeper не применяет собственную контрольно-кассовую технику. Фискализация расчётов осуществляется через платёжного партнёра — ООО «ЮKassa» (партнёр Яндекс.Денег), который является зарегистрированным платёжным агентом и оператором электронных средств платежа.

2.3. ООО «ЮKassa» обеспечивает:

- Регистрацию ККТ и работу с оператором фискальных данных (ООО «Эвотор» / Taxcom).
- Формирование кассовых чеков по каждому платежу Пользователя.
- Передачу фискальных данных в ФНС России.
- Отправку электронных чеков Пользователям на адрес электронной почты, указанный при оплате.

2.4. Таким образом, Платформа не является плательщиком, а действует как посредник, обеспечивающим передачу данных о расчётах в ОФД через платёжного партнёра.

## 3. ОБРАБОТКА ПЛАТЕЖЕЙ ЧЕРЕЗ YOOKASSA

3.1. Интеграция с ООО «ЮKassa» обеспечивает следующую цепочку фискализации:

1. Пользователь инициирует оплату в приложении VaultKeeper.
2. Платформа формирует платёжный запрос к API YooKassa.
3. Пользователь вводит данные банковской карты и подтверждает платёж (3-D Secure).
4. YooKassa обрабатывает платёж и автоматически формирует фискальный документ.
5. Фискальный документ передаётся через ОФД в ФНС.
6. Электронный чек отправляется Пользователю на email.
7. YooKassa уведомляет Платформу об успешном проведении платежа и статусе фискализации.

3.2. Платформа VaultKeeper получает от YooKassa следующие данные по каждому платежу:

- Идентификатор платежа (payment_id).
- Статус фискализации (fiscalization_status).
- Номер фискального документа (fiscal_document_number).
- Фискальный признак (fiscal_sign).
- Ссылка на электронный чек.

3.3. Указанные данные хранятся в системе VaultKeeper в течение 5 (пяти) лет и доступны по запросу контролирующих органов.

## 4. ВЫПЛАТА ВОЗНАГРАЖДЕНИЙ ХОСТАМ

4.1. Выплата вознаграждений Хостам является отдельным расчётным сценарием, который подлежит фискализации в соответствии с 54-ФЗ.

4.2. Процесс выплаты:

1. По итогам расчётного периода (месяц) Платформа формирует ведомость выплат Хостам.
2. Через API YooKassa инициируется массовый платёж (bank_card_payout).
3. YooKassa фискализирует каждую выплату и отправляет чек Хосту на email.
4. Данные о выплате и чеке сохраняются в системе VaultKeeper.

4.3. Для Хостов, зарегистрированных как индивидуальные предприниматели (ИП), допускается безналичный перевод на расчётный счёт с последующим самостоятельным документооборотом. В этом случае чек формируется по оплате услуг Пользователями, а выплата ИП оформляется как банковский перевод.

4.4. Для Хостов — самозанятых (плательщиков НПД) применяется особый порядок, описанный в разделе 5.

## 5. НАЛОГОВЫЙ РЕЖИМ НПД (САМОЗАНЯТЫЕ) ДЛЯ ХОСТОВ

5.1. Хосты, зарегистрированные как плательщики налога на профессиональный доход (НПД, «самозанятые»), являются наиболее распространённой категорией участников сети.

5.2. Ставки налога НПД:

- 4 % — при получении дохода от физических лиц (Пользователей).
- 6 % — при получении дохода от юридических лиц (если Платформа выступает как юридическое лицо).

5.3. VaultKeeper обеспечивает:

- **Формирование чеков НПД:** при каждой выплате самозанятому Хосту Платформа автоматически генерирует чек через приложение «Мой налог», передавая данные через API Федеральной налоговой службы.
- **Расчёт и удержание налога:** сумма налога (4–6 %) рассчитывается автоматически и удерживается из суммы выплаты Хосту. Налог перечисляется в бюджет ФНС самозанятым самостоятельно через приложение «Мой налог». Платформа информирует Хоста о сумме налога.
- **Предоставление информации:** Платформа предоставляет Хосту ежемесячный отчёт о доходах для декларирования в приложении «Мой налог».

5.4. Платформа не несёт ответственности за своевременность и правильность уплаты налога самозанятым Хостом. Платформа выступает как информационный посредник и агента платежа.

## 6. СИСТЕМА АВТОМАТИЧЕСКОЙ ГЕНЕРАЦИИ ДОКУМЕНТОВ

6.1. Для обеспечения соответствия требованиям 54-ФЗ и налогового законодательства в платформе VaultKeeper реализована автоматическая система генерации и хранения документов:

**Модуль фискализации (fiscalization-service):**
- Интеграция с API YooKassa для получения статусов фискализации.
- Хранение фискальных данных в защищённой базе данных с шифрованием (AES-256).
- Ретроспективный аудит: возможность повторного запроса данных о фискализации за любой период.

**Модуль генерации чеков НПД (npd-receipt-service):**
- Интеграция с API ФНС (протокол v2) для формирования чеков самозанятых.
- Автоматический расчёт суммы налога и формирование отчётов.
- Обработка ошибок и повторные попытки отправки чеков при временных сбоях API ФНС.

**Модуль финансовой отчётности (finance-report-service):**
- Генерация ежемесячных финансовых отчётов для Платформы и Хостов.
- Подготовка данных для налоговой отчётности (КНД 1111021, книга учёта доходов для НПД).
- Экспорт отчётов в форматах PDF, Excel и CSV.

**Модуль хранения документов (document-archive):**
- Хранение всех финансовых и фискальных документов в течение 5 лет.
- Предоставление документов по запросу контролирующих органов (система авторизованного доступа).
- Ведение журнала доступа к документам (аудит-трейл).

6.2. Все модули разработаны на языке Rust в рамках единой архитектуры микросервисов и проходят регулярное тестирование, включая тесты интеграции с API YooKassa и ФНС.

## 7. ЗАКЛЮЧЕНИЕ

7.1. Платформа VaultKeeper обеспечивают полное соответствие требованиям 54-ФЗ путём делегирования фискализации расчётов платёжному партнёру ООО «ЮKassa».

7.2. Выплата вознаграждений Хостам фискализируется через механизмы YooKassa, а для самозанятых Хостов дополнительно обеспечивается генерация чеков НПД через API ФНС.

7.3. Система автоматической генерации документов обеспечивает хранение фискальных данных, подготовку отчётности и аудитоспособность операций.

7.4. Платформа рекомендует всем Хостам проконсультироваться с налоговым консультантом относительно индивидуальных налоговых обязательств.
